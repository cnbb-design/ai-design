# Domain Taxonomy for Eloquent JavaScript

## Categories

- **fundamentals**: Core language primitives — values, types, operators, expressions, statements, bindings
- **control-flow**: Program structure — conditionals, loops, branching, iteration patterns
- **functions**: Function definition, invocation, scope, closures, arrow functions, recursion
- **data-structures**: Arrays, objects, strings, maps, sets, JSON, destructuring
- **higher-order-programming**: Higher-order functions, functional patterns, abstraction, composition
- **object-oriented-programming**: Prototypes, classes, inheritance, encapsulation, polymorphism, interfaces
- **error-handling**: Bugs, debugging, exceptions, try/catch, strict mode, testing
- **text-processing**: Regular expressions, pattern matching, string manipulation
- **modularity**: Modules, packages, namespaces, dependency management, ES modules
- **asynchronous-programming**: Callbacks, promises, async/await, event loop, timers
- **language-implementation**: Parsing, evaluation, syntax trees, interpreters
- **web-platform**: Networks, protocols, HTTP, URLs, browsers, web standards
- **dom**: Document Object Model, element creation/manipulation, tree traversal, querying
- **events**: Event handling, listeners, propagation, delegation, keyboard/mouse events
- **graphics**: Canvas API, 2D drawing, SVG, coordinate systems, animation, paths
- **http-forms**: HTTP methods, fetch API, forms, request/response, AJAX
- **server-side**: Node.js, file system, streams, server creation, command-line tools
- **application-architecture**: State management, components, data flow, routing, full-stack patterns

## Tiers

- **foundational**: No prerequisites from this source (Ch 0-2 concepts: values, types, operators, bindings, basic control flow)
- **intermediate**: Requires foundational concepts (Ch 3-11, 13-15, 17-18, 20 concepts: functions, data structures, OOP, async, DOM, events)
- **advanced**: Requires intermediate concepts (Ch 6 advanced OOP, Ch 12 language implementation, Ch 16/19/21 project-level architecture)

## Notation Conventions

### Code
- Use backtick-wrapped inline code for all JavaScript identifiers: `let`, `const`, `function`
- Use fenced code blocks for multi-line examples
- Preserve the source's code examples exactly

### Types
- Refer to types as: number, string, boolean, undefined, null, object, symbol, bigint
- Use lowercase for type names in prose

### Operators
- Use source notation: `+`, `-`, `*`, `/`, `%`, `===`, `!==`, `&&`, `||`, `!`
- For comparison: distinguish `==` (loose) from `===` (strict)

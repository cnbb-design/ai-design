# Quick Definition

The primary JavaScript reference resources are MDN Web Docs (comprehensive web technology reference), Node.js Docs (server-side API), and ExploringJS.com (book series by the author).

# Core Definition

"If you have a question about JavaScript, I can recommend the following online resources: MDN Web Docs: cover various web technologies such as CSS, HTML, JavaScript, and more. An excellent reference. Node.js Docs: document the Node.js API. ExploringJS.com: My other books cover various aspects of web development." (Ch. 8, &sect;8.3).

# Prerequisites

Foundational concept with no prerequisites.

# Key Properties

1. MDN Web Docs: comprehensive web technology reference
2. Node.js Docs: server-side API documentation
3. ExploringJS.com: Deep JavaScript, TypeScript, Node.js shell scripting
4. Compatibility tables: MDN, caniuse.com, compat-table, node.green

# Construction / Recognition

Look up any JavaScript API or feature on MDN for comprehensive documentation with examples and browser compatibility tables.

# Context & Application

MDN is the go-to reference for JavaScript developers. Node.js Docs are essential for server-side development.

# Common Errors

- **Error**: Relying on outdated or unofficial JavaScript reference sources.
  **Correction**: Use MDN Web Docs as the primary reference; it's community-maintained and up-to-date.

# Common Confusions

- **Confusion**: Not knowing which reference to use for browser vs. server APIs.
  **Clarification**: Use MDN for browser APIs and language features; Node.js Docs for Node-specific APIs.

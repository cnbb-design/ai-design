# Quick Definition
The Deno Standard Library (`@std`) is a collection of audited, stable packages published on JSR that provide common utilities -- from ULID generation to async helpers -- with autogenerated documentation supplemented by editorial overrides.

# Core Definition
The Deno Standard Library is a set of packages under the `@std` scope, published on JSR (JavaScript Registry). These packages provide commonly needed functionality that complements the runtime's built-in APIs. Examples referenced in the KV documentation include `@std/ulid` (for generating ULIDs used as KV key parts) and `@std/async` (for async utilities like `delay`).

Each `@std` package page is autogenerated from JSR metadata by the `scripts/generate_std_docs.ts` script. The generated pages include frontmatter (title, description, version, stability), a stability banner, the package overview (HTML from JSR), and optionally an "Additional Examples" section sourced from editorial override files in `_overrides/<package>.md`. Pages also support persistent custom content blocks preserved across regeneration cycles using `<!-- custom:start -->` / `<!-- custom:end -->` markers.

# Prerequisites
- Understanding of the Deno runtime (deno)
- Familiarity with JSR (JavaScript Registry) for package imports

# Key Properties
1. **@std scope on JSR** -- All standard library packages are published under `@std` on JSR
2. **Import via jsr:** -- Packages imported as `jsr:@std/<package>` (e.g., `jsr:@std/ulid`)
3. **Autogenerated docs** -- Documentation pages are generated from JSR metadata with supplemental editorial overrides
4. **Stability tracking** -- Each package has a stability level indicated in frontmatter
5. **Override system** -- Custom examples and guides can be added via `_overrides/<package>.md` files without being overwritten by regeneration

# Construction / Recognition
- Import ULID: `import { ulid } from "jsr:@std/ulid";`
- Import async delay: `import { delay } from "jsr:@std/async@1/delay";`
- Regenerate docs: `deno task generate:std-docs`

# Context & Application
The standard library provides a vetted, consistent set of utilities that avoid the need for third-party dependencies for common tasks. Its tight integration with the documentation pipeline ensures that package documentation stays current with upstream changes on JSR while allowing editorial enhancements.

# Common Errors
1. Using outdated import URLs -- standard library packages should be imported from `jsr:@std/` not legacy `deno.land/std/` URLs
2. Editing autogenerated documentation outside of custom blocks -- changes will be overwritten on next regeneration

# Common Confusions
1. **Standard library vs. built-in APIs** -- The `Deno` namespace provides built-in APIs; the standard library (`@std`) provides additional utility packages that must be imported
2. **JSR vs. npm** -- Standard library packages live on JSR (`jsr:@std/`), not npm

# Quick Definition

Nine principles of page layout that operationalize good typography: start with body text, divide foreground from background, use the smallest visible increments, try both options, be consistent, relate elements to each other, keep it simple, imitate what you like, and don't fear white space.

# Core Definition

Butterick presents nine maxims: "Successful typography requires you to pay attention to the whole, not just the parts."

1. **Decide first how the body text will look.** Four decisions — point size, line length, line spacing, and font — "have the biggest influence on the legibility of the text."
2. **Divide the page into foreground and background.** "Don't let the background elements upstage the foreground elements."
3. **Make adjustments with the smallest visible increments.** "The difference between not enough and too much can be small."
4. **When in doubt, try it both ways.** "Don't try to resolve typographic decisions with logic."
5. **Be consistent.** "Things that are the same should look the same. Things that look different should actually be different."
6. **Relate each new element to existing elements.** "The page is like a jigsaw puzzle that becomes more constrained with each new piece."
7. **Keep it simple.** "If you think you need three colors and five fonts, think again."
8. **Imitate what you like.** "Why reinvent the wheel?"
9. **Don't fear white space.** "Work outward from the text, not inward from the page edges."

# Key Properties

1. **Body text first**: Everything else follows from the body text decisions.
2. **Foreground/background**: Conscious hierarchy of page elements.
3. **Smallest increments**: Typography thrives on fine details.
4. **Visual judgment**: "No substitute for making samples... and getting a visual reaction."
5. **Consistency as structure**: Typography communicates structure and hierarchy.
6. **Constraint accumulates**: Each element constrains subsequent choices.
7. **Simplicity**: Fewer fonts, fewer colors.
8. **Imitation as learning**: Studying good examples builds visual judgment.
9. **White space is positive**: Not wasted — it supports readability.

# Construction / Recognition

## To Construct/Create:
1. Start with body text settings (point size, line spacing, line length, font).
2. Identify foreground elements; subordinate everything else.
3. Make one small change at a time; evaluate visually.
4. Compare options side by side rather than deciding abstractly.
5. Apply consistent formatting to similar elements.

## To Identify/Recognise:
1. Good layout: feels organized, readable, consistent.
2. Bad layout: feels random, cluttered, or disproportionate.

# Examples

**Example 1** (maxims-of-page-layout): Maxim #5 — "Things that are the same should look the same. Things that look different should actually be different. Without consistent treatment of similar elements, the document will come across as random and meandering."

**Example 2** (maxims-of-page-layout): Maxim #9 — "A lot of mediocre typography results from a perceived need to fill space."

# Common Errors

- **Error**: Starting layout design with decorative elements, logos, or headings.
  **Correction**: "Decide first how the body text will look" — everything else follows.

# Common Confusions

- **Confusion**: Good layout requires mathematical ratios or formal grid systems.
  **Clarification**: "Don't try to resolve typographic decisions with logic. There's no substitute for making samples of two options and getting a visual reaction."

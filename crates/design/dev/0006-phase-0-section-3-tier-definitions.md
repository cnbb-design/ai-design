# Phase 0 - Section 3 - Tier Definitions

## 3. Tier Definitions

### 3.1 Tier Criteria

| Tier | Slug | Prerequisites | Knowledge Type | User Can... |
|------|------|--------------|----------------|-------------|
| Foundational | `foundational` | None within this knowledge base | Vocabulary, atomic concepts, first principles | Name it, recognise it, define it |
| Intermediate | `intermediate` | Requires foundational concepts | Applied knowledge, pattern recognition, procedures | Use it, combine it, diagnose simple problems |
| Advanced | `advanced` | Requires intermediate concepts | Systematic thinking, cross-domain integration, architectural decisions | Design systems from it, critique holistically, make principled trade-offs |

### 3.2 Tier Assignment by Category

| Category | Foundational | Intermediate | Advanced |
|----------|-------------|-------------|---------|
| `visual-elements` | All concepts (point, line, plane, shape, texture, value) | — | — |
| `visual-perception` | Gestalt principles, figure-ground | Preattentive processing, visual search | Computational models of perception, persistent homology formalisms |
| `design-principles` | Definitions of each principle | Application patterns, diagnostic use | Cross-domain integration, principle trade-offs |
| `colour-theory` | Colour properties (hue, saturation, lightness), colour wheel, primary models | Palette construction, colour contrast (Itten), colour temperature, HSL vs. OKLCH | Perceptual uniformity mathematics, CIE colour science, CIEDE2000, dynamic palette generation from tokens |
| `typography` | Type anatomy, type classification, basic hierarchy | Modular scales, pairing, measure, leading | Fluid type systems (Utopia), variable font axes, optical compensation, typographic grids |
| `layout-composition` | Grid basics, alignment, whitespace | Column/modular grids, responsive strategies, CSS layout models | Intrinsic design, container queries, Every Layout primitives, subgrid |
| `interaction-design` | Affordances, signifiers, mapping, feedback | State design, microinteractions, progressive disclosure | Mental model alignment, Gulf of Execution/Evaluation, system conceptual models |
| `motion-design` | Easing fundamentals, duration conventions | Choreography, enter/exit patterns, meaningful motion | Physics-based animation, scroll-driven animations, cubic-bezier mathematics |
| `information-architecture` | Content hierarchy, navigation types | Task-driven vs. data-model IA, information scent | Cross-system IA, scalable taxonomy design |
| `data-visualisation` | Encoding channels, basic chart types | Perceptual effectiveness, annotation, responsive data vis | Tufte's principles, Cleveland & McGill rankings, complex interactive vis |
| `imagery` | Image types, basic framing | Photography direction, icon systems, illustration style | Image as system component, art direction across breakpoints |
| `design-systems` | What tokens are, atomic design levels | Token architecture (primitive/semantic/component), component API design | Multi-brand theming, DTCG specification, governance, Figma↔code pipelines |
| `accessibility` | WCAG principles, contrast ratios | ARIA patterns, focus management, keyboard nav | Cognitive accessibility, systematic accessibility auditing, inclusive design as architectural constraint |
| `visual-identity` | Logo anatomy, clear space | Identity systems, visual language consistency | Multi-platform identity systems, brand as design system extension |
